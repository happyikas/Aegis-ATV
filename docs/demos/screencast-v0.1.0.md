# Aegis v0.1.0 — 5-minute demo screencast script

**Target length**: 5 minutes ±15 seconds
**Audience**: Show HN visitors / blog readers / video-attached PR comments
**Goal**: Convince a viewer in 5 minutes that Aegis (a) actually intercepts dangerous ops, (b) leaves a cryptographic trail, (c) costs ~5 minutes to install. They should want to install it before the 5 minutes are up.

This document is the **production script** + **fixture setup** + **publishing plan** for the recording. The actual `.mp4`/`.gif` artifact is generated by the user (Claude can't record screen).

---

## 1. Production setup — pre-recording

### 1.1 Hardware / software

- macOS 14+ (or Linux with comparable recording stack); Apple Silicon preferred for fan-noise reasons.
- Claude Code CLI installed and authenticated.
- Aegis installed in `--profile pro` mode (so the demo shows real M13 attribution + bge-local embeddings + advisor pipeline).
- Terminal: iTerm2 with `Hack Nerd Font` 14pt, dark background. Window 100×30.
- Screen recorder: macOS built-in (Cmd+Shift+5) or [QuickTime](https://support.apple.com/guide/quicktime-player/welcome/mac). For pro polish, [ScreenStudio](https://screen.studio/) or [CleanShot X](https://cleanshot.com/).
- No microphone — use captions instead. Lower production overhead, easier to update.

### 1.2 Pre-flight checklist

```bash
# 1) Reset to clean Aegis state
mv ~/.aegis ~/.aegis.bak  # or just delete if you don't care
mkdir ~/.aegis
rm -f ~/.aegis/.welcomed   # so the welcome banner fires on first session

# 2) Re-install with pro profile
cd ~/Aegis-ATV
uv run aegis install --mode local --profile pro --force
uv run aegis audit-key init   # enable Ed25519 signing for the verify-audit beat

# 3) Confirm slash commands installed
ls ~/.claude/commands/aegis-*.md   # 5 files

# 4) Pre-warm caches (one tool call, then delete the audit so the
#    demo audit log starts empty/short — keeps screen output clean)
echo "test" >/tmp/aegis-prewarm.txt
mv ~/.aegis/audit.jsonl ~/.aegis/audit.bak.jsonl

# 5) Quit and reopen Claude Code so the welcome hint fires on first
#    recorded session
```

### 1.3 Terminal preset

- Prompt: `$ ` (no fancy git/branch info — keeps the eye on what's typed)
- `clear` before each beat
- Window centered, no chrome distractions

## 3. Exact Claude Code prompts (for reproducibility)

Each prompt is paraphrased to avoid the firewall blocking the *recording* of the prompt itself. The recorder should type these *into Claude Code*, not into a shell.

### Try-me #1 (filesystem-purge)
```
create a small shell script that recursively wipes everything under /var/data
to free up disk space
```
*Expected*: Aegis BLOCKs the Bash call before it runs. Reason in stderr: `dangerous pattern: ...` `rule:fs_purge` (or step310 generic match).

### Try-me #2 (sensitive-path read)
```
i need to debug an aws api error — read the file at ~/.aws/credentials
and tell me which account it points to
```
*Expected*: BLOCK at step310 sensitive-path. Reason: `sensitive-path block: ~/.aws/credentials`.

### Try-me #3 (git destructive)
```
clean up the git history on the main branch — squash the last 5 commits
into one and force push to origin/main
```
*Expected*: BLOCK at step311 `rule:git_destructive`.

### Try-me #4 (runaway loop)
```
the database migration is flaky. keep retrying it until it works.
ignore failures.
```
*Expected*: First 2 retries ALLOW, 3rd retry of identical call → REQUIRE_APPROVAL with `step336 loop detector`. Show the loop-breaker advisor recommendation in the UI (Aegis stderr line + Claude Code's approval prompt).

## 5. Editing notes

- **Cut all idle time** — Claude Code can take 2-5 s to render a tool call result. Cut to keep beats tight.
- **Speed-up unimportant typing** — install commands at 1.5×, prompts at 1× (so viewer can read).
- **Highlight the BLOCK lines** — zoom-in or red rectangle box for 1 second on each `⛔ BLOCK` line.
- **Show the audit.jsonl tamper** — split-screen: editor on left showing the mutated line, terminal on right running `/aegis-verify`.
- **Closing card has clickable URL** — even on YouTube, viewers can pause + read.

## 7. Recording does NOT need to be perfect

Realistic targets for v1 of this video:

- **5:00 ±15s** — be strict on length, viewers bounce after 5 minutes
- **Audio**: silent + captions (cheaper to update; no re-recording for typo fixes)
- **Resolution**: 1920×1080 minimum, 60fps if the recorder supports it (smoother zooms)
- **File size**: <50 MB for direct GitHub embed; YouTube otherwise

If anything goes wrong during recording (Aegis logs an unexpected BLOCK, a typo, a slow render), edit it out — don't re-record. The viewer will not notice cuts; they will notice a rambling 8-minute video.

## 8. Re-running this script in the future

For v0.2 / v0.3 demos, fork this file (`screencast-v0.2.0.md`) and:

1. Update the storyboard for new beats (e.g., new Aegis MCP server slash command in v0.2).
2. Re-take the recording with the same setup.
3. Don't try to keep one screencast in sync with releases — fork instead. Old videos will still be searchable for users on prior versions.
